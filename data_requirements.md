# 데이터 요구사항 및 기준 정의 (Data Requirements)

본 문서는 AI 수강신청 보조 서비스 운영을 위해 필요한 데이터의 구조와 분류 기준을 정의합니다. 현재는 하드코딩된 더미 데이터를 기반으로 동작하며, 추후 엑셀 원본 데이터를 파싱할 때 본 기준을 적용합니다.

## 1. 과목 분류 기준 (Course Categories)

과목은 크게 전공과 교양으로 나뉘며, 세부 태그를 통해 AI 추천을 용이하게 합니다.

### 전공 (Major)
- **전공필수 (Major Required)**: 졸업을 위해 반드시 이수해야 하는 과목.
- **전공선택 (Major Elective)**: 전공 심화 또는 타 전공과의 융합을 위한 선택 과목.

### 교양 (General Education)
교양 과목은 학생들이 자신의 관심사에 맞게 검색할 수 있도록 세부 분류(태그)가 필수적입니다.
- **분류 체계 예시**:
  - `[인문/사회]`: 철학, 심리학, 역사, 경제
  - `[자연/과학]`: 수학, 물리, 환경, 컴퓨터 개론
  - `[어학]`: 영어 회화, 제2외국어
  - `[예체능]`: 미술, 체육, 음악

## 2. 강의실 및 동선 데이터 (Location Data)

시간표 생성 시, 연강(연달아 듣는 강의)인 경우 건물 간 이동 거리를 계산하여 경고를 발생시킵니다.
- **건물 노드 예시**:
  - `A동(본관)`
  - `B동(공학관)`
  - `C동(인문관)`
- **동선 경고 기준**:
  - 이동 시간(휴식 시간) 15분 이내에 도달할 수 없는 거리인 경우 (예: `B동` ↔ `C동` 간 거리가 도보 20분일 때 연강이면 경고 플래그).

## 3. 학생 개인 데이터 모델 (User Profile)

학생의 잔여 학점과 이수 학점을 계산하기 위한 구조입니다.
- **이수 학점 총합** (예: 130학점 중 현재 110학점)
- **분야별 잔여 학점**:
  - 전공 필수 잔여 학점
  - 전공 선택 잔여 학점
  - 교양 필수 잔여 학점

## 4. 데이터베이스 테이블 구조 (SQLite Schema)

### `courses` 테이블
| 컬럼명 | 데이터 타입 | 설명 | 예시 |
|--------|-------------|------|------|
| id | Integer | PK | 1 |
| code | String | 과목 코드 | CS101 |
| title | String | 과목명 | 파이썬 프로그래밍 |
| professor | String | 담당 교수 | 홍길동 |
| credits | Integer | 학점 | 3 |
| category | String | 분류 (전필, 전선, 교양 등) | 전필 |
| tags | String | AI 검색용 키워드(쉼표 구분) | "코딩,기초,파이썬,실습" |
| location | String | 건물 및 호실 | 공학관 301호 |
| schedule | String | 시간 (요일 및 교시) | 월1,수2 |

### `cart` 테이블
| 컬럼명 | 데이터 타입 | 설명 |
|--------|-------------|------|
| id | Integer | PK |
| course_id | Integer | `courses` 테이블의 FK |
| user_id | Integer | 사용자 ID |
